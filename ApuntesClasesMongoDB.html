Apuntes Clases MongoDB

Conceptos básicos de MongoDB

  1. Estructura de MongoDB:
    • Base de datos: Es el contenedor principal que almacena las colecciones.
    • Colección: Equivalente a una tabla en bases de datos relacionales (SQL). Una colección almacena documentos.
    • Documento: Equivalente a un registro o fila en una tabla SQL. Un documento es un conjunto de pares clave-valor en formato JSON (BSON en MongoDB).

  1. Comandos básicos en MongoDB:
    • use <nombre_bd>: Selecciona una base de datos para trabajar con ella. Si no existe, la crea automáticamente.
    • db: Muestra el nombre de la base de datos actual.
    • show collections: Muestra todas las colecciones en la base de datos actual.
    • db.dropDatabase(): Elimina la base de datos actual.
    • exit: Cierra la sesión de MongoDB.

  1. Inserción de datos:
    • insertOne(): Inserta un solo documento en una colección.
    • insertMany(): Inserta varios documentos en una colección.
    • Si la colección no existe, MongoDB la crea automáticamente al insertar el primer documento.

  1. Consulta de datos:
    • find(): Muestra todos los documentos en una colección. Equivalente a SELECT * en SQL.
    • find({filtro}): Filtra documentos según condiciones específicas. Por ejemplo:
      Esto devuelve solo los campos
      nombre y edad de los usuarios con nombre "Kain".
      db.usuarios.find({ nombre: "Kain" }, { nombre: 1, edad: 1 })
    • findOne(): Devuelve el primer documento que coincide con el filtro.
    • count(): Cuenta el número de documentos que coinciden con un filtro.

  1. Actualización de datos:
    • updateOne(): Actualiza el primer documento que coincide con el filtro.
    • updateMany(): Actualiza todos los documentos que coinciden con el filtro.
    • Operadores de actualización:
      • $set: Establece un nuevo valor para un campo.
        db.usuarios.updateOne({ edad: 20 }, { $set: { edad: 21 } })
      • $inc: Incrementa el valor de un campo.
        db.usuarios.updateMany({ edad: 12 }, { $inc: { edad: 3 } })
      • $unset: Elimina un campo de un documento.

  1. Reemplazo de documentos:
    • replaceOne(): Reemplaza un documento completo por uno nuevo.

  1. Eliminación de datos:
    • deleteOne(): Elimina el primer documento que coincide con el filtro.
    • deleteMany(): Elimina todos los documentos que coinciden con el filtro.

  1. Operadores de consulta:
    • $eq: Igual a.
    • $ne: No igual a.
    • $gt: Mayor que.
    • $gte: Mayor o igual que.
    • $lt: Menor que.
    • $lte: Menor o igual que.
    • $in: Encuentra documentos donde el valor del campo está en un conjunto específico.
    • $nin: Encuentra documentos donde el valor del campo no está en un conjunto específico.
    • $and: Combina múltiples condiciones con un AND lógico.
    • $or: Combina múltiples condiciones con un OR lógico.
    • $exists: Filtra documentos donde un campo existe o no.
    • $expr: Permite comparar campos dentro de un documento. Por ejemplo:
      Esto devuelve los usuarios cuyo saldo es mayor que su deuda.
      db.usuarios.find({ $expr: { $gt: ["$saldo", "$deuda"] } })

Sintaxis y formato en MongoDB

  • MongoDB utiliza una sintaxis basada en JSON (JavaScript Object Notation).
  • Los filtros y operaciones se escriben entre llaves {}.
  • Los operadores comienzan con el símbolo $ (por ejemplo, $set, $inc, $gt).

Ejemplos prácticos

  1. Crear una colección e insertar documentos:
    use mi_bd
db.usuarios.insertOne({ nombre: "Kain", edad: 20 })
db.usuarios.insertMany([{ nombre: "Ana", edad: 25 }, { nombre: "Luis", edad: 30 }])

  1. Consultar documentos:
    db.usuarios.find({ edad: { $gt: 20 } })

  1. Actualizar documentos:
    db.usuarios.updateMany({ edad: 20 }, { $set: { edad: 21 } })

  1. Eliminar documentos:
    db.usuarios.deleteOne({ nombre: "Kain" })

Índices en Bases de Datos

  1. Definición de Índices:
    • Un índice es un objeto que se crea en la base de datos para clasificar los registros en función de uno o varios atributos.
    • Su propósito principal es optimizar las consultas, haciendo que el motor de la base de datos las ejecute de manera más eficiente.
    • Los índices permiten evitar recorrer toda la tabla en tiempo de ejecución, ya que previamente se ha ordenado la información según el atributo o atributos seleccionados.

  1. Índices Compuestos:
    • Un índice compuesto es aquel que se crea sobre dos o más atributos de una tabla.
    • Esto es útil cuando las consultas frecuentes involucran múltiples columnas.

  1. Operaciones con Índices:
    • Creación de Índices: Se pueden crear índices sobre una colección o tabla, especificando los atributos relevantes.
    • Eliminación de Índices: Los índices pueden eliminarse cuando ya no son necesarios.
    • Listado de Índices: Es posible listar todos los índices creados dentro de una colección o tabla.

  1. Uso de Índices en Consultas:
    • El motor de la base de datos utiliza automáticamente los índices para optimizar las consultas.
    • También es posible forzar a una consulta a usar un índice específico mediante la sintaxis adecuada (por ejemplo, en MongoDB, se puede especificar el índice con .hint()).

Agregaciones en MongoDB

  1. Concepto de Agregaciones:
    • Las agregaciones permiten realizar operaciones complejas sobre los datos, estructurando la consulta en etapas o fases.
    • Cada etapa realiza una operación específica, como filtrar, agrupar, ordenar o calcular valores.

  1. Etapas de una Agregación:
    • Filtrado: Se seleccionan los documentos que cumplen ciertas condiciones (por ejemplo, filtrar usuarios mayores de 18 años).
    • Agrupación: Los documentos se agrupan según un atributo (por ejemplo, agrupar usuarios por ciudad).
    • Cálculos: Se realizan operaciones como sumas, promedios o conteos sobre los grupos.
    • Ordenación: Los resultados se ordenan según un criterio específico.

  1. Comparación con SQL:
    • Las agregaciones en MongoDB son equivalentes a consultas complejas en SQL, pero se estructuran en etapas en lugar de una sola consulta.

Operadores en MongoDB

  1. Operadores de Filtrado:
    • El operador $match se utiliza para filtrar documentos, similar a la cláusula WHERE en SQL.
    • Ejemplo: Filtrar usuarios por edad o ciudad.

  1. Operadores de Agrupación:
    • El operador $group permite agrupar documentos según un atributo y realizar cálculos sobre los grupos.
    • Ejemplo: Contar cuántos usuarios hay en cada ciudad.

  1. Operadores de Renombrado:
    • El operador $project permite renombrar atributos o modificar la estructura de los documentos de salida.
    • Ejemplo: Combinar dos atributos (nombre y apellido) en uno solo (nombre completo).

  1. Operadores de Unión (Lookup):
    • El operador $lookup realiza una operación similar a un JOIN en SQL, permitiendo combinar datos de dos colecciones.
    • Ejemplo: Unir la colección de usuarios con la colección de pedidos.

Almacenamiento de Resultados

  1. Uso de $merge:
    • El operador $merge permite almacenar los resultados de una consulta en una nueva colección.
    • Si la colección ya existe, los resultados se combinan con los datos existentes.
    • Ejemplo: Almacenar el resultado de una agregación (como el conteo de usuarios por ciudad) en una nueva colección.

Eficiencia en Consultas

  1. Problema de Tablas Grandes:
    • Cuando una tabla tiene muchos registros y atributos, las consultas pueden volverse lentas.
    • Los índices ayudan a optimizar estas consultas al reducir el tiempo de búsqueda.

  1. Ejemplo de Índice por Ciudad:
    • Si se crea un índice sobre el atributo "ciudad", la base de datos no necesita recorrer toda la tabla para encontrar los registros de una ciudad específica.
    • En su lugar, accede directamente a los registros ordenados en el índice.

Resumen

  • Los índices son herramientas esenciales para optimizar consultas en bases de datos, ya que permiten ordenar y acceder a los datos de manera más eficiente.

  • Las agregaciones en MongoDB permiten realizar consultas complejas estructuradas en etapas, similares a las operaciones de filtrado, agrupación y ordenación en SQL.

  • Los operadores como $match, $group, $project y $lookup son fundamentales para manipular y analizar datos en MongoDB.

  • El uso de índices y agregaciones mejora significativamente el rendimiento de las consultas, especialmente en tablas con grandes volúmenes de datos.
